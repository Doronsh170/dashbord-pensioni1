# דשבורד חשיפות חיסכון

כלי שמאפשר לכל אחד לראות בקליק את הרכב החיסכון שלו —
ללא הרשמה, כל הנתונים נשארים מקומית במכשיר.

**גרסה:** v3.40 — כניסה מפושטת: בחירה לפי גוף, לא לפי הקלדה.
**נתונים:** מבוססים על מאגרים רשמיים (גמל נט, פנסיה נט, ביטוח נט) —
תקופת דיווח אחרונה: **202603**.

## מה השתנה ב-v3.40

- **כותרת חדשה:** "תראה איך הכסף שלך מושקע" — בלי מונחים פיננסיים בכניסה.
- **סריג חברות:** 8 הגופים הגדולים בישראל (מנורה מבטחים, הראל, הפניקס, כלל, מגדל, מיטב, מור, אלטשולר שחם) — קליק במקום הקלדה.
- **מחוון שלבים:** המשתמש רואה מראש שזה זרימה של 3 צעדים (חברה ← מסלול ← הרכב).
- **חיפוש משני:** שדה ההקלדה והפילטרים נשמרו, אבל ירדו אל מתחת לסריג כאופציה לשימוש מתקדם.
- **תאי מידע מקוצרים:** מ-4 ל-2 (פרטיות + מקור) — שני המסרים החיוניים.

## מבנה הריפו

```
.
├── index.html              ← הדשבורד (single-file HTML)
├── funds_unified.json      ← מאגר הנתונים המאוחד
├── README.md               ← הקובץ הזה
├── .gitignore
└── tools/                  ← אינו מוגש לציבור — כלי עזר לעדכון נתונים
    ├── README.md
    ├── build.py            ← בונה funds_unified.json מתוך הקבצים בסורס
    └── source/             ← אקסלים מקוריים של הרגולטור
        ├── גמל.xlsx
        ├── פנסיה.xlsx
        └── ביטוח.xlsx
```

## פריסה ל-GitHub Pages

1. `git push` לריפו ב-GitHub.
2. Settings ← Pages ← Source: main / root.
3. הדף עולה בכתובת `https://<username>.github.io/<repo>`.

הקובץ היחיד שהמשתמש פוגש הוא `index.html`. הוא טוען את `funds_unified.json`
באותה תיקייה. אין צורך בשרת.

## עדכון נתונים

ראה `tools/README.md` — תהליך החלפת האקסלים והרצת build.py.

## הערות

- הכל מקומי. הזנת היתרה נשמרת ב-localStorage בלבד.
- אין הרשמה, אין שרת, אין מעקב.
- הנתונים משקפים את הדיווחים הציבוריים של המסלולים — לא תשואה אישית.
